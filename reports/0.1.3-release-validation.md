# 0.1.3 Release Validation

## Result

The v0.1.3 checkpoint-resilience release passed the complete 585-test source suite on Linux with Python 3.12. The wheel and source distribution build from the release tree, strict metadata validation passes, and an isolated wheel install exposes version 0.1.3 through the distribution metadata and Python package. Console entry-point help and version checks also pass against the built wheel with the declared runtime dependencies supplied from the validated source environment.

Validation date: 2026-07-15

Release base: `b18a2e2dd6e5e84cf4a9ce1c82095590aa19052b` (`v0.1.2`)

## Scope

This release changes only the policy applied after resumable ASR checkpoint I/O failures. The low-level checkpoint writer still propagates file and directory durability errors. The transcription finalizer now reports an I/O warning and continues to transactional output publication instead of marking an otherwise completed file as failed. Non-I/O checkpoint failures remain fatal so invalid checkpoint data and implementation errors are not hidden.

The regression test injects a real directory-sync failure into checkpoint persistence, then verifies that the job remains successful, the transcript and manifest are published, the checkpoint success counter is not incremented, and the warning is reported. A paired test verifies that an invalid checkpoint payload still prevents publication. The existing low-level test continues to verify that the checkpoint writer itself propagates directory-sync failures.

This release does not change model loading, audio decoding, segmentation, VAD, ASR generation, batching, alignment, transcript rendering, output formats, public API contracts, or serialized schema versions.

The real TorchCodec tests require the CUDA 13 runtime libraries installed with PyTorch 2.11.0 to be visible to the dynamic loader. Without that environment path, four TorchCodec tests fail before package code executes because `libnvrtc.so.13` is unavailable. With the validated PyTorch and CUDA library directories on `LD_LIBRARY_PATH`, all 27 audio-backend tests and all 585 package tests pass.

## Runtime Validation Decision

No real-model CUDA transcription benchmark was repeated for v0.1.3 because inference behavior and performance-sensitive paths are unchanged. The retained v0.1.0 and v0.1.1 transcript, accuracy, alternate-model, and performance evidence remains applicable. The v0.1.2 durability evidence remains applicable to the unchanged low-level writers and transactional output publication.

## Release Assessment

The v0.1.3 tree preserves strict checkpoint durability diagnostics without allowing a recovery artifact to suppress a completed primary transcript. All local source, typing, packaging, installed-wheel, and deterministic failure-path gates pass within the validated Linux scope.
